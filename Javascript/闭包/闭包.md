# 闭包

## 闭包的定义

```txt
内部函数有权访问外部函数的变量，即使外部函数已执行完毕，但内部函数对外部函数变量的引用依然存在。那么该内部函数及其周围环境以及被引用的变量统称为闭包。
```

## 闭包的特点

```txt
优点：
1.数据持久化：被闭包内部函数引用的变量不会被JS垃圾回收机制回收，外部函数执行完毕，数据依然存在于内存中。
2.数据私有化：外部环境无法直接访问闭包内部的变量，需要通过闭包暴露的接口来访问。
3.函数柯里化：将多参数函数转化为单一参数的多个函数。
注意事项：
1.由于闭包引用的变量无法被JS垃圾回收机制回收，所以该变量会常驻内存，故滥用闭包会导致内存泄露，可在变量使用完毕时手动将该变量设置为null，释放内存。
2.闭包中内部函数引用的外部变量是对变量的引用，而非独立的数据，所以外部环境中该变量发生变化时，内部引用的变量也会发生变化。
```

## 代码实现
```js
			function outerFn() {
				const outerVar = { name: "吕威鹏", age: 18 };
				function innerFn() {
					console.log("引用的外部函数变量>>>", outerVar);
				}
				return innerFn;
			}
			const getVar = outerFn();
			getVar(); // 这里虽然 outerFn 执行完毕，但 innerFn 依然能访问 outerVar
```
